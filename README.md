## Purpose
- Centralize practice and task submissions for the web development curriculum.
- Provide consistent structure and documentation for each task.

## Conventions
- Each task folder should include:
  - `index.html` ‚Äî main entry point
  - `styles.css` ‚Äî styles
  - `script.js` ‚Äî JavaScript (if needed)
  - `README.md` ‚Äî task-specific instructions, demo, and notes
- Use semantic HTML, accessible markup (alt attributes, roles), and responsive styles.

## How to Use
1. Open a task folder in your editor (e.g., VS Code).
2. Run a local static server or use Live Server to preview pages.
3. Update or add your work in a new task folder following the structure above.

## Development Tips
- Keep commits small and descriptive (e.g., `task1: add responsive header`)
- When adding images, place them in an `assets/` or `images/` subfolder.
- Include a screenshot in the task `README.md` for quick previews.
